## 🎯 Overview

### Goal
Improve supervisor user experience by:
- Enforcing single-project assignment per supervisor
- Removing unnecessary features from supervisor role
- Simplifying navigation and workflows
- Preventing accidental data modifications across projects

### Scope
- **Role**: Supervisor only
- **Focus**: Large-scale projects with multiple supervisors
- **Authorization Model**: Full authorization within assigned project, no cross-project access

---

## ❌ Current Issues

### Issue 1: Multi-Project Visibility
**Problem:**
- Supervisors can see ALL projects (AEP01, Main Construction, etc.)
- Supervisors can create/edit sites across ANY project
- No project-based isolation

**Impact:**
- Risk of supervisor accidentally modifying wrong project
- Confusion when switching between projects
- Security concern: supervisors seeing data from other projects

**Example:**
```
Supervisor A (assigned to AEP01) can:
❌ See sites from "Main Construction" project
❌ Create sites in "Main Construction"
❌ Edit sites in "Main Construction"
```

---

### Issue 2: Unnecessary CRUD Permissions
**Problem:**
- Supervisors can DELETE sites
- Site deletion should be admin-only function
- Accidental deletion can cause data loss

**Impact:**
- Risk of accidental site deletion
- Loss of historical data (progress logs, inspections, etc.)
- No proper authorization control

---

### Issue 3: Project Assignment Not Enforced
**Problem:**
- UserModel doesn't have `project_id` field
- Project assignment is informal, not enforced at database level
- No validation to ensure supervisor works on assigned project only

**Impact:**
- No single source of truth for project assignment
- Difficult to audit which supervisor manages which project
- Cannot enforce project-based filtering at query level

---

### Issue 4: Too Many Tabs (Future Consideration)
**Problem:**
- 7 tabs in supervisor navigator
- MaterialTracking tab is logistics function, not supervisor's daily task

**Impact:**
- Cluttered interface
- Cognitive overload
- Supervisors confused about which tab to use

**Note:** Keeping MaterialTracking for now, will deliberate later.

---

### 1. UserModel.ts

**Before:**
```typescript

  static table = 'users';

  static associations: Associations = {
    roles: { type: 'belongs_to', key: 'role_id' },

  @field('role_id') roleId!: string;
```

**After:**
```typescript

    roles: { type: 'belongs_to', key: 'role_id' },

  @field('project_id') projectId?: string; // NEW

}
```

```
Phase Filters:
[All Phases] [Design] [Approvals] [Mobilization] [Procurement]
[Interface] [Site Prep] [Construction] [Testing] [Commissioning]
[SAT] [Handover]
```
❌ **Problem:** Supervisors don't work on "Design" or "Approvals" at site level

**Proposed Change:**
Show only **site-relevant phases** for supervisors:

```
Phase Filters (Supervisor View):
[All Phases] [Site Prep] [Construction] [Testing]
```

- ❌ Design (office/engineering work)
