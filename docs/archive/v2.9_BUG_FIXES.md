# v2.9 Bug Fixes - Association Name Corrections

**Date:** 2025-11-21
**Issue:** Hindrances, Site Inspections, and Reports History failing to load

---

## Root Cause

The Q.on() queries were using incorrect association names. The models define their associations with **singular** names ('site'), but the queries were using **plural** ('sites').

### Model Associations (Correct Names):

| Model | Association Name | Collection Table |
|-------|-----------------|------------------|
| HindranceModel | `site` (singular) | hindrances |
| SiteInspectionModel | `site` (singular) | site_inspections |
| DailyReportModel | `site` (singular) | daily_reports |
| ItemModel | `sites` (plural) | items |

**Note:** ItemModel is the exception - it uses 'sites' (plural) as the association name.

---

## Fixes Applied

### 1. HindranceReportScreen.tsx (Line 90)

**Before:**
```typescript
const queryConditions = [
  Q.where('reported_by', supervisorId),
  Q.on('sites', 'project_id', projectId), // WRONG - causes error
];
```

**After:**
```typescript
const queryConditions = [
  Q.where('reported_by', supervisorId),
  Q.on('site', 'project_id', projectId), // CORRECT - singular 'site'
];
```

**Error Fixed:**
```
Query on 'hindrances' joins with 'sites', but HindranceModel does not have
associations={} defined for 'sites'
```

## Technical Learning

**WatermelonDB Rule:** Q.on() must use the EXACT association name defined in the Model's `static associations` object, not the collection/table name.

**Example:**
```typescript
// In Model:
static associations: Associations = {
  site: { type: 'belongs_to', key: 'site_id' },  // Association name: 'site'
};

// In Query - MUST match association name:
Q.on('site', 'project_id', projectId)  // ✅ CORRECT

// NOT the collection name:
Q.on('sites', 'project_id', projectId)  // ❌ WRONG
```
