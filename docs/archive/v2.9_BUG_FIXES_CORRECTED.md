# v2.9 Bug Fixes - Model Association Corrections

**Date:** 2025-11-21
**Issue:** Hindrances, Site Inspections, and Reports History failing to load
**Error:** `no such table: site (code 1 SQLITE_ERROR)`

---

## Root Cause Analysis

The issue was in the **Model association definitions**, NOT in the query code.

### The Problem:

WatermelonDB models had associations defined with **singular names** ('site') but the actual database table is **'sites'** (plural).

When using `Q.on('site', ...)`:
1. WatermelonDB looks for an association named 'site' ✅
2. WatermelonDB tries to JOIN with a table named 'site' ❌ (doesn't exist - table is 'sites')

### The SQL Error:

```sql
select "hindrances".* from "hindrances"
join "site" on "site"."id" = "hindrances"."site_id"
-- ERROR: no such table: site
```

**Should be:**
```sql
select "hindrances".* from "hindrances"
join "sites" on "sites"."id" = "hindrances"."site_id"
```

---

## The Correct Fix

The association name in models must match the **table name**, not some arbitrary name.

### Key Insight:

In WatermelonDB, when you use `Q.on('association_name', ...)`, it:
1. Looks up the association by name
2. **Uses that association name as the table name to JOIN**

Therefore: **Association name = Table name** (for belongs_to relationships)

## Technical Learning

### WatermelonDB Association Naming Rules:

1. **For `belongs_to` relationships:**
   - Association name MUST match the target **table name**
   - Example: If joining with 'sites' table → association must be named 'sites'

2. **For `has_many` relationships:**
   - Association name is the target table name (plural)
   - Example: A site has many items → association is 'items'

3. **The `key` field:**
   - Specifies the foreign key column in the current table
   - Example: `{ type: 'belongs_to', key: 'site_id' }` means this table has a 'site_id' column

### Correct Pattern:

```typescript
// hindrances table has site_id column
// hindrances belongs to sites table

class HindranceModel extends Model {
  static table = 'hindrances';

  static associations: Associations = {
    sites: {                    // ← Must match table name 'sites'
      type: 'belongs_to',       // ← This table belongs to sites
      key: 'site_id'            // ← Foreign key in hindrances table
    },
  };

  @field('site_id') siteId!: string;
}

// Query usage:
Q.on('sites', 'project_id', projectId)  // ← Use association name 'sites'
```
