## Why Snackbar First?

### Current Problem with Alert.alert():
- ❌ **Blocks UI** - User cannot interact with app while alert is showing
- ❌ **Breaks Workflow** - User must dismiss alert before continuing
- ❌ **No Undo** - Once dismissed, message is gone forever
- ❌ **Inconsistent UX** - Different on iOS vs Android
- ❌ **Not Mobile-First** - Desktop-style modal interrupts mobile flow

### Benefits of Snackbar:
- ✅ **Non-blocking** - User can continue working while message shows
- ✅ **Auto-dismiss** - Disappears after 3-4 seconds automatically
- ✅ **Action Support** - Can add "Undo" or "View" actions
- ✅ **Stacking** - Multiple messages queue nicely
- ✅ **Consistent** - Same UX on iOS and Android
- ✅ **Mobile-First** - Bottom-anchored, thumb-friendly

## Implementation Steps

### Phase 1: Create Snackbar Infrastructure (Day 1-2)

#### Step 1.1: Create Types
**File:** `src/components/Snackbar/types.ts`

```typescript
export type SnackbarType = 'success' | 'error' | 'warning' | 'info';

export interface SnackbarMessage {
  id: string;
  message: string;
  type: SnackbarType;
  duration?: number;
  action?: {
    label: string;
    onPress: () => void;
  };
}

export interface SnackbarContextValue {
  showSnackbar: (message: string, type?: SnackbarType, action?: SnackbarMessage['action']) => void;
  hideSnackbar: () => void;
}
```

#### Step 1.2: Create Snackbar Provider
**File:** `src/components/Snackbar/SnackbarProvider.tsx`

**Features:**

#### Replacement Pattern:

**OLD (Alert.alert):**
```typescript
import { Alert } from 'react-native';

const handleSave = async () => {
  try {
    await saveItem();
    Alert.alert('Success', 'Item saved successfully!');
  } catch (error) {
    Alert.alert('Error', 'Failed to save item');
  }
};

const handleDelete = () => {
  Alert.alert(
    'Delete Item',
    'Are you sure?',
    [
      { text: 'Cancel', style: 'cancel' },
      { text: 'Delete', onPress: confirmDelete, style: 'destructive' }
    ]
  );
};
```

**NEW (Snackbar + Dialog):**
```typescript
import { useSnackbar } from '../../components/Snackbar/useSnackbar';
import { useState } from 'react';
import ConfirmDialog from '../../components/Dialog/ConfirmDialog';

const { showSnackbar } = useSnackbar();
const [showDeleteDialog, setShowDeleteDialog] = useState(false);

const handleSave = async () => {
  try {
    await saveItem();
    showSnackbar('Item saved successfully!', 'success');
  } catch (error) {
    showSnackbar('Failed to save item', 'error');
  }
};

const handleDelete = () => {
  setShowDeleteDialog(true);
};

const confirmDelete = async () => {
  setShowDeleteDialog(false);
  await deleteItem();
  showSnackbar('Item deleted', 'success');
};

// In JSX:
<ConfirmDialog
  visible={showDeleteDialog}
  title="Delete Item"
  message="Are you sure you want to delete this item? This action cannot be undone."
  confirmText="Delete"
  cancelText="Cancel"
  onConfirm={confirmDelete}
  onCancel={() => setShowDeleteDialog(false)}
/>
```

## Risks & Mitigation

### Risk 1: Breaking Existing Functionality
**Mitigation:**
- Test each screen after migration
- Keep git history for easy rollback
- Migrate module-by-module, not all at once

### Risk 2: Snackbar Covering Important UI
**Mitigation:**
- Use bottom-anchored position (default)
- Auto-dismiss after 4 seconds
- Allow swipe-to-dismiss gesture

### Risk 3: Dialog Not Closing Properly
**Mitigation:**
- Use controlled component pattern (visible={state})
- Always set state to false in onCancel/onConfirm
- Add backdrop dismiss handler

### Risk 4: TypeScript Type Errors
**Mitigation:**
- Define strict types upfront
- Use TypeScript's utility types
- Test with strict mode enabled
