### 5. src/planning/components/SimpleSiteSelector.tsx
**Changed:** HOC pattern → RxJS observable subscription

**Before:**
```typescript
const enhance = withObservables([], () => ({
  sites: database.collections.get('sites').query(),
}));
const SimpleSiteSelector = enhance(Component);
```

**After:**
```typescript
useEffect(() => {
  const subscription = database.collections
    .get('sites')
    .query()
    .observe()
    .subscribe(setSites);
  return () => subscription.unsubscribe();
}, []);
```

**Why:** withObservables HOC doesn't work with functional components, caused runtime error

## 🎯 Key Workflow Changes

### Before v1.7 (Broken Workflow)
```
1. Admin creates Project
2. ❌ Supervisor creates Site (BLOCKING PLANNER!)
3. Planner creates WBS Items
```

**Problem:** Planners blocked until supervisors create sites

---

### After v1.7 (Correct Workflow)
```
1. Admin creates Project
2. ✅ Planner creates Site (optional supervisor assignment)
3. Planner creates WBS Items
4. Planner assigns Supervisor to Site
5. Supervisor manages assigned Site
```

**Solution:** Planners work independently, assign supervisors when ready

## 🎓 Lessons Learned

### 1. Workflow Dependency Analysis
**Always map user workflows before implementing features.**
- We discovered supervisor dependency late
- Should have been caught during initial design
- Now fixed with proper workflow

### 2. Tab Ordering Matters
**UI element order should match user mental model.**
- Original order was arbitrary (implementation order)
- Logical order dramatically improves UX
- Small change, big impact

### 3. Documentation is Code
**Keep documentation in sync with code changes.**
- Updated 9 documentation files
- Each serves specific purpose
- Comprehensive docs = easier maintenance

### 4. Observable Patterns
**Use appropriate React patterns for data fetching.**
- withObservables HOC = Class components only
- .observe().subscribe() = Functional components
- Know your tools!
