## 🎯 Overview

This document maps each Manager Dashboard metric to its exact data source (database table + field). Use this to understand how to update data to see changes reflected in the dashboard.

---

## 📊 Section 1: Project Overview & KPIs

### Overall Project Completion %
**Source:** Calculated from hybrid formula
**Formula:** `(60% × Weighted Items Progress) + (40% × Milestone Progress)`
**Database Tables:**
- `items` table → `completed_quantity`, `planned_quantity`, `weightage`
- `milestone_progress` table → `progress_percentage`

**How to Update:**
1. Update item progress: Go to **Supervisor app** → Site → Items → Update completed quantity
2. Update milestone progress: Use **Milestone Management tab** → View Progress → Update per site

**Code Location:** `ManagerDashboardScreen.tsx:415-487` (calculateHybridProgress)

---

### Sites Status (On Schedule / Delayed)
**Source:** Calculated from site target dates
**Database Tables:**
- `sites` table → `target_date`, `start_date`

**How to Update:**
1. Update site dates in database directly OR
2. Through Admin/Supervisor screens when creating/editing sites

**Calculation:** Sites with `target_date >= today` are "on schedule"
**Code Location:** `ManagerDashboardScreen.tsx:309-317` (calculateStats)

---

### Budget Utilization %
**Source:** Calculated from purchase orders
**Formula:** `(Total PO Value / Project Budget) × 100`
**Database Tables:**
- `projects` table → `budget` (Project total budget)
- `purchase_orders` table → `po_value`, `status`

**How to Update:**
1. **Project Budget:** Update `projects.budget` field directly in database
2. **PO Values:** Create purchase orders through appropriate screen (future feature)
3. **For testing:** Insert PO records directly into `purchase_orders` table

**Code Location:** `ManagerDashboardScreen.tsx:489-507` (calculateBudgetUtilization)

---

### Open Issues / Hindrances
**Source:** Hindrances with status = 'open'
**Database Tables:**
- `hindrances` table → `status`, `site_id`

**How to Update:**
1. Go to **Supervisor app** → Site → Hindrances → Add hindrance
2. Status must be 'open' (not 'resolved' or 'closed')

**Code Location:** `ManagerDashboardScreen.tsx:331-336`

---

### Critical Path Items
**Source:** Items marked as critical_path = true
**Database Tables:**
- `items` table → `critical_path`, `status`, `site_id`

**How to Update:**
1. Update `items.critical_path` field to `true` in database
2. Items with delays will be counted as "at risk"

**Code Location:** `ManagerDashboardScreen.tsx:338-350`

---

### Pending Approvals
**Source:** NOT YET IMPLEMENTED
**Status:** Shows 0 (placeholder)
**Future:** Will count items/documents pending manager approval

---

### Deliveries Status
**Source:** Purchase orders with delivery dates
**Database Tables:**
- `purchase_orders` table → `expected_delivery_date`, `status`

**How to Update:**
1. Create PO records with delivery dates
2. POs with `expected_delivery_date < today` and `status != 'delivered'` count as delayed

**Code Location:** `ManagerDashboardScreen.tsx:354-371`

---

### Upcoming Milestones (Next 30 Days)
**Source:** Milestone progress with planned end dates
**Database Tables:**
- `milestone_progress` table → `planned_end_date`, `status`
- `milestones` table → `milestone_name`

**How to Update:**
1. Go to **Milestone Management tab** → View Progress (per site)
2. Set `planned_end_date` for each site's milestone
3. Dates within 30 days from today will be counted

**Code Location:** `ManagerDashboardScreen.tsx:373-401`

## 🔧 Section 2: Engineering Progress (PM200 + DOORS + RFQ)

### PM200 Design Completion %
**Source:** Milestone progress for PM200
**Database Tables:**
- `milestones` table → `milestone_code = 'PM200'`
- `milestone_progress` table → `milestone_id`, `progress_percentage`

**How to Update:**
1. Go to **Milestone Management tab**
2. Find PM200 (Engineering & Design) milestone
3. Click "View Progress" → Update progress for each site
4. Overall % = average across all sites weighted by milestone weightage

**Code Location:** `ManagerDashboardScreen.tsx:1194-1206`

---

### DOORS Packages
**Source:** DOORS packages table
**Database Tables:**
- `doors_packages` table → `project_id`, `status`, `equipment_name`
- `doors_requirements` table → `package_id`, `compliance_status`

**How to Update:**
1. **Via DOORS screens** (if implemented in another role)
2. **Direct database insert:**
```sql
INSERT INTO doors_packages (id, project_id, package_code, equipment_name, status, ...)
VALUES (newid(), 'your_project_id', 'PKG-001', 'Main Transformer', 'approved', ...);
```

**Statuses:** 'draft', 'under_review', 'approved', 'rejected'
**Code Location:** `ManagerDashboardScreen.tsx:1208-1259`

---

## 🏗️ Section 3: Site Progress (All Sites Comparison)

## 💰 Section 5: Financial Summary

### Project Budget
**Source:** Projects table
**Database Tables:**
- `projects` table → `budget`

**How to Update:**
1. Update `projects.budget` field directly in database
2. Set realistic value (e.g., 10,000,000 for ₹1 Crore)

---

### Budget Allocated (BOM Total)
**Source:** Sum of all BOM costs
**Database Tables:**
- `boms` table → `project_id`, `total_cost`

**How to Update:**
1. **BOM Management tab** → Add BOM → Add items with costs
2. Total cost = sum of (quantity × unit_cost) for all items

**Code Location:** `FinancialReportsScreen.tsx:116-133`

---

### Budget Spent (Actual Cost)
**Source:** Sum of PO values (delivered + closed)
**Database Tables:**
- `purchase_orders` table → `po_value`, `status IN ('delivered', 'closed')`

**How to Update:**
1. Create POs with status = 'delivered' or 'closed'
2. Only these statuses count toward actual spending

**Code Location:** `FinancialReportsScreen.tsx:184-207`

---

### Contract Value
**Source:** Same as Project Budget
**Assumption:** Contract value = Total budget
**Database Tables:**
- `projects` table → `budget`

---

### Profitability Metrics
**Calculations:**
- **Projected Profit** = Contract Value - Actual Cost
- **Profit Margin %** = (Projected Profit / Contract Value) × 100

**Code Location:** `FinancialReportsScreen.tsx:216-218`

---

### Cost Breakdown by Category
**Source:** BOM items grouped by category
**Database Tables:**
- `bom_items` table → `category`, `total_cost`

**Categories:** 'material', 'labor', 'equipment', 'subcontractor'

**How to Update:**
1. BOM Management → Add items with specific categories
2. Set `category` field when creating BOM items

**Code Location:** `FinancialReportsScreen.tsx:149-167`

## 📝 Database Quick Insert Examples

### Add a Purchase Order:
```sql
INSERT INTO purchase_orders (
  id, project_id, po_number, vendor_id, po_value,
  status, po_date, created_at, updated_at, sync_status
) VALUES (
  'po_' || hex(randomblob(8)),
  'your_project_id',
  'PO-2025-001',
  'vendor_id',
  100000,
  'delivered',
  strftime('%s', 'now') * 1000,
  strftime('%s', 'now') * 1000,
  strftime('%s', 'now') * 1000,
  'synced'
);
```

### Add DOORS Package:
```sql
INSERT INTO doors_packages (
  id, project_id, package_code, equipment_name,
  status, created_at, updated_at, sync_status
) VALUES (
  'pkg_' || hex(randomblob(8)),
  'your_project_id',
  'PKG-001',
  'Main Control Panel',
  'approved',
  strftime('%s', 'now') * 1000,
  strftime('%s', 'now') * 1000,
  'synced'
);
```

### Add RFQ:
```sql
INSERT INTO rfqs (
  id, project_id, rfq_number, equipment_description,
  status, issue_date, closing_date, created_at, updated_at, sync_status
) VALUES (
  'rfq_' || hex(randomblob(8)),
  'your_project_id',
  'RFQ-2025-001',
  'HT Switchgear Panels',
  'issued',
  strftime('%s', 'now') * 1000,
  strftime('%s', 'now', '+30 days') * 1000,
  strftime('%s', 'now') * 1000,
  strftime('%s', 'now') * 1000,
  'synced'
);
```

### Update Item Progress:
```sql
UPDATE items
SET completed_quantity = 50,
    updated_at = strftime('%s', 'now') * 1000
WHERE id = 'your_item_id';
```

### Update Milestone Progress:
```sql
UPDATE milestone_progress
SET progress_percentage = 75,
    status = 'in_progress',
    updated_at = strftime('%s', 'now') * 1000
WHERE milestone_id = 'your_milestone_id'
  AND site_id = 'your_site_id';
```

## 🎯 Testing Workflow

### To Test Dashboard Completely:

1. **Setup Project Budget:**
   ```sql
   UPDATE projects SET budget = 10000000 WHERE id = 'your_project_id';
   ```

2. **Add Some Items Progress:**
   - Use Supervisor app to update 5-10 items
   - Set various completed_quantity values

3. **Update Milestone Progress:**
   - Go to Milestone Management
   - For each milestone (PM100-PM700), click "View Progress"
   - Update progress for 2-3 sites (set to 20%, 50%, 80%, etc.)

4. **Add Financial Data:**
   - Create 2-3 BOMs with items
   - Insert 2-3 PO records (see SQL above)

5. **Add Test Data:**
   - Insert 1-2 DOORS packages
   - Insert 1-2 RFQs
   - Add hindrances through Supervisor app

6. **Refresh Dashboard:**
   - Pull down to refresh
   - All metrics should now show non-zero values
